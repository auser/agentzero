01

Talk to your codebase. Safely.

AgentZero connects to a local model and gives it tools to read files, search code, and run commands — but every tool call passes through policy. Dangerous operations require your explicit approval.

~/project
you> are there any leaked credentials in this repo?
  [tool: search] pattern="password=" … ok (890 B)
  [tool: read] config/database.yml … ok (240 B)

agentzero> Found one issue. config/database.yml line 8 has a
hardcoded password. The value was redacted from this response.

  $ agentzero vault add postgres db-password

you> run the full security audit
  [APPROVE shell: agentzero run repo-security-audit?] (y/n) y
  Scanning 42 files… 3 findings. Report → .agentzero/audit/report.md

02

One line to secure your editor's AI.

AgentZero runs as an MCP server. Point Claude Code, Cursor, or Zed at it and every tool call goes through policy, gets audited, and respects your project's security rules.

.claude/settings.json
{
  "mcpServers": {
    "agentzero": {
      "command": "agentzero",
      "args": ["mcp"]
    }
  }
}

Five tools, fully gated

read_file · list_directory · search_files · write_file · run_command

Every call goes through path validation, sensitive path blocking, and emits a structured audit event. No tool call is invisible.

03

What we built. And what we didn't.

A small, auditable core — not a framework. Every security invariant enforced in code, not documentation.

Deny by default

Unknown permissions denied. Unknown data classifications fail closed. The agent proves access before acting.

Secret handles

Models see handle://vault/github/token. Never raw values. Exposed only at execution time, only after policy.

PII redaction

Scanned before remote model calls. Emails, tokens, keys replaced with typed placeholders. Automatic.

Content provenance

Tool output labeled untrusted. The model knows what is data and what is instruction. Injection does not escalate.

Encrypted at rest

AES-256-GCM + Argon2id. Vault, sessions, audit logs. Per-line encryption so logs stay appendable.

Local first

Ollama, llama.cpp, vLLM, LM Studio. Your data stays on your machine. Remote calls require explicit policy.
