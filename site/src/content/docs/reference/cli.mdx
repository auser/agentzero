## `agentzero init`

Initialize a new project.

```bash
agentzero init [--private]
```

| Flag | Description |
|------|-------------|
| `--private` | Use private-by-default policy (network denied, local-only routing) |

## `agentzero chat`

Start an interactive chat session.

```bash
agentzero chat [OPTIONS]
```

| Flag | Description |
|------|-------------|
| `--local` | Use local models only |
| `-m, --model` | Model name (default: llama3.2) |
| `--stream` | Stream tokens as they arrive |
| `-p, --provider` | Provider: ollama, llama-cpp, vllm, lm-studio |
| `--url` | Custom server URL |
| `--resume` | Resume a previous session by ID |
| `--encrypt` | Encrypt audit and session files |

## `agentzero mcp`

Start the MCP server for editor integration.

```bash
agentzero mcp
```

Reads JSON-RPC 2.0 from stdin, writes to stdout.

## `agentzero serve`

Start the ACP server.

```bash
agentzero serve
```

## `agentzero run`

Run an installed skill.

```bash
agentzero run <name>
```

Routes by skill runtime declared in `SKILL.md`:

- **WASM skills** — creates a session, checks policy, builds a sandbox profile, and executes the `.wasm` module through the wasmtime engine. WASM is compiled in by default (disable with `--no-default-features`); requires `wasm_execution = "allow"` or `"require_approval"` in `policy.yml`.
- **Host-supervised skills** — runs the entrypoint script (default `run.sh`) on the host with `ShellCommand` capability check. Policy-gated and audited.
- **Scanner skills** (with `patterns.toml`) — runs the security scanner against the current directory.
- **Instruction-only skills** — displays the `SKILL.md` content.

## `agentzero install`

Install a skill from a local path, GitHub owner/repo, or git URL.

```bash
agentzero install <source>
```

The source is auto-detected:

| Format | Example | Action |
|--------|---------|--------|
| `owner/repo` | `auser/my-skill` | Download latest GitHub release |
| GitHub URL | `https://github.com/auser/my-skill` | Download latest GitHub release |
| Git URL | `https://gitlab.com/user/repo` | Clone via git |
| Local path | `/path/to/skill` or `./my-skill` | Copy directory |

GitHub installs verify SHA-256 checksums from the release notes. All install methods update `.agentzero/skills.lock`.

## `agentzero publish`

Package and publish a skill to a GitHub release.

```bash
agentzero publish [path] [OPTIONS]
```

| Flag | Description |
|------|-------------|
| `--repo` | Target GitHub repository (owner/repo). Auto-detected from git remote if omitted. |
| `--tag` | Release tag (default: `v<version>` from SKILL.md) |

Requires `GITHUB_TOKEN` environment variable. Creates a GitHub release, uploads the tarball, and includes the SHA-256 checksum in the release notes.

## `agentzero doctor`

System diagnostic. Reports crate status, project initialization, policy rules, skills, vault, sessions, providers, and WASM sandbox status (compiled-in, policy setting, WASM skill count).

```bash
agentzero doctor
```

## `agentzero history`

List past chat sessions.

```bash
agentzero history
```

## `agentzero policy status`

Show active policy rules.

```bash
agentzero policy status
```

## `agentzero audit tail`

Show recent audit events.

```bash
agentzero audit tail [-c, --count N]
```

## `agentzero vault`

Manage encrypted secrets.

```bash
agentzero vault list
agentzero vault add <provider> <name>
agentzero vault get <provider> <name>
agentzero vault remove <provider> <name>
```
